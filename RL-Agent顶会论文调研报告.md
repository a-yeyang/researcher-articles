## 一、先打地基：5 分钟看懂这个领域在说什么

如果你是小白，先把下面几个词搞懂，后面 10 篇论文就都能看懂了。

### 1. 强化学习（RL, Reinforcement Learning）
**一句话**：让模型像"打游戏练级"一样，通过"试错 + 拿分（奖励）"自己学会做对事情。
- **类比**：训练小狗。狗做对动作 → 给零食（奖励）；做错 → 没零食。多来几次，狗就学会了。
- 对大模型来说：模型输出一段内容 → 有个"打分器"给分（答案对不对 / 任务成没成）→ 模型朝"高分方向"调整自己。

### 2. 智能体（Agent）
**一句话**：能"自己多步行动、调用工具、和环境反复打交道"来完成任务的 AI，而不是问一句答一句。
- **类比**：普通大模型像"考试答题"（给题→给答案，一锤子买卖）；智能体像"实习生干活"（先查资料→调用工具→看结果→不对就重试→最后交付），中间有**很多步**。
- 典型动作：调搜索引擎、跑一段代码、点网页按钮、操作手机界面。

### 3. "RL + Agent" 到底指什么？（⚠️ 最容易搞混的点）
这个领域其实分两类，**面试和读论文时千万别混**：

| 类型 | 干什么 | 是否多步/用工具 | 代表 |
|---|---|---|---|
| **推理 RL（LLM-RL）** | 用 RL 让模型"想得更对"（如解数学题），**单步**输出答案 | ❌ 不与外部环境交互 | GRPO、DeepSeek-R1 |
| **智能体 RL（Agentic RL）** | 用 RL 训练模型**多步**地用工具/查搜索/点网页/操作系统 | ✅ 反复和环境交互 | 本报告的 10 篇 |

**你要的"RL+Agent"= 第二类 Agentic RL。** 但第一类是它的"地基"（见第二节），必须知道。

### 4. 三个最常出现的训练算法（各记一句话即可）
- **PPO**（近端策略优化）：最经典的 RL 训练算法，稳但要额外训一个"评委网络（critic）"，比较吃显存。
- **DPO**（直接偏好优化）：不用训评委，直接用"好答案 vs 坏答案"成对数据来调模型，简单高效。
- **GRPO**（组相对策略优化）：DeepSeek 提出的"PPO 简化版"，去掉评委网络，靠"一组答案互相比"来打分。**现在几乎所有 Agentic RL 都在用它**，所以一定要认识。

### 5. 两个常被提到的训练范式
- **RLHF**（基于人类反馈的 RL）：用人类偏好训"打分器"，再用它训模型。ChatGPT 就靠这个变"听话"。
- **RLVR**（基于可验证奖励的 RL）：奖励不靠人，而靠"能自动判对错"的信号（如数学答案对不对、代码能不能跑过测试）。**Agentic RL 大多用这个**，因为任务成没成往往能自动判定。

### 6. 反复出现的"考场"（基准/环境），认个脸
- **WebShop / ALFWorld / ScienceWorld**：模拟"网购 / 家务 / 科学实验"的文字交互环境，测 agent 多步决策。
- **AitW（Android-in-the-Wild）/ AndroidWorld / OSWorld / WebArena**：测 agent 操作手机、电脑、网页的真实界面任务。
- **AIME / GSM8K / MATH**：数学竞赛题，测"推理 + 用工具算"。
- **HotpotQA / NQ / TriviaQA**：需要"多跳查资料"的问答，测搜索 agent。

## 三、2024 年 前五（按引用排）

### 🥇 1. ETO —《Trial and Error: Exploration-Based Trajectory Optimization for LLM Agents》
- **会议**：ACL 2024 主会　|　**引用**：181　|　**代码**：https://github.com/Yifan-Song793/ETO

**核心思想（大白话）**：以前训 agent 只给它看"标准答案录像"（成功轨迹），它学得死板。ETO 说：**让 agent 自己去试、去失败，然后把"失败的做法"和"成功的做法"摆在一起对比着学**——就像人"吃一堑长一智"。

**怎么做**：
1. 先用监督学习（行为克隆）做一个"基础 agent"；
2. 让它在环境里实际去做任务，**收集一堆失败轨迹**；
3. 把"失败轨迹 ↔ 成功轨迹"配成对；
4. 用 **DPO 损失**让模型"喜欢成功的、讨厌失败的"；
5. 这套流程可**多轮迭代**，越练越强。

**实验怎么做**：在 **WebShop（网购）、ScienceWorld（科学实验）、ALFWorld（家务）** 三个交互环境测试；对比只用成功轨迹的 SFT 等基线，ETO 成功率明显更高，且在"没见过的任务"上更稳。

**为什么重要**：把"从失败中学习"这个朴素想法，用 DPO 干净利落地实现，成了后续 agent 训练的常用套路。

### 🥈 2. ArCHer —《ArCHer: Training Language Model Agents via Hierarchical Multi-Turn RL》
- **会议**：ICML 2024　|　**引用**：180　|　**代码**：https://github.com/YifeiZhou02/ArCHer

**核心思想（大白话）**：多轮任务的难点是"**功劳/过错算给谁**"——任务最后失败了，到底是哪一步说错了话？ArCHer 用"**分层**"解决：高层管"每一句话好不好"，低层管"每个字怎么说"。

**怎么做**：两层 RL 嵌套——
- **高层**：用**离线（off-policy）TD 学习**训一个"价值评委"，评估"说完这句话后局面好不好"（话语级，credit assignment）；
- **低层**：用**在线（on-policy）token 级策略梯度**，在高层给的目标下决定"这句话具体每个字怎么生成"。

**实验怎么做**：在多个多轮 agent 任务（如 **Twenty Questions 二十问、猜词、对话购物**等）测试；和纯在线方法（PPO）、其他离线方法对比。

**主要结果**：样本效率比纯在线方法**提升约 100 倍**（作者说法，在 Twenty Questions 上最明显：PPO 要十万级样本，ArCHer 千级样本就达到更好水平），最终性能也超过其他离线方法。

**为什么重要**：给"长程多轮 + 奖励稀疏"的 agent 训练提供了高效的分层框架。

### 🥉 3. DigiRL —《DigiRL: Training In-The-Wild Device-Control Agents with Autonomous RL》
- **会议**：NeurIPS 2024　|　**引用**：177　|　**代码**：https://github.com/DigiRL-agent/digirl

**核心思想（大白话）**：想让 AI 真的会"用手机"（点按钮、滑屏、填表）。光给它看人类操作录像（模仿学习）不够，因为真实手机界面千变万化。DigiRL 让 agent **自己在真手机环境里点来点去试错**，用 RL 学会操作。

**怎么做**：把一个预训练的**视觉-语言模型（VLM）**用两阶段 RL 微调——
1. **离线 RL**：先用已有数据离线训；
2. **离线转在线 RL**：再让它在真实 Android 环境里实际操作、拿成败反馈继续学。
（还设计了"自动判断任务成没成"的评估器来给奖励。）

**实验怎么做**：在 **AitW（Android-in-the-Wild）** 真实安卓任务上测（如在 Google 搜索、购物、看新闻等）。

**主要结果**：一个 **1.5B 的小 VLM，用 RL 后成功率从 17.7% 飙到 67.2%（绝对提升 49.5 个百分点）**，超过用静态人类示范做监督微调的版本，也超过当时基于 GPT-4V / Gemini 的 agent——**且不需要额外人工标注**。

**为什么重要**：证明了"小模型 + 在真实环境里 RL 试错" 能把 GUI 操作能力拉到 SOTA，是手机/电脑操作 agent 的代表作。

### 4. StepCoder —《StepCoder: Improve Code Generation with RL from Compiler Feedback》
- **会议**：ACL 2024　|　**引用**：待补（API 限流未取到）　|　**代码**：https://github.com/Ablustrund/APPS_Plus　|　复旦大学

**核心思想（大白话）**：用 RL 训模型写代码，奖励信号就是"**代码能不能编译/跑过测试**"（编译器反馈）。但有两个坑：(1) 复杂需求要写很长的代码，模型一上来全写对太难，RL 探索不到正反馈；(2) 测试用例覆盖不全时，拿"没被执行到的代码"去优化是无效的。StepCoder 两招破解。

**怎么做**：
- **CCCS（代码补全课程）**：把"从零写整段代码"这个难任务，拆成"**先补全一小段→逐步补全更多**"的由易到难课程，让模型循序渐进；
- **FGO（细粒度优化）**：**只对真正被测试执行到的代码 token 计算梯度**，没执行到的部分屏蔽掉，让优化更准。
- 还构建了 **APPS+ 数据集**（人工核验过单元测试，保证奖励信号干净）。

**实验怎么做**：在 **APPS+、MBPP、HumanEval** 等代码基准上测；对比标准 PPO 等 RL 基线。

**主要结果**：在这些基准上优于直接用 PPO 的方法，尤其在复杂题目上探索更有效。

**为什么重要**：是"RL 训代码能力"里把"探索难 + 奖励噪声"两个工程问题讲清楚的代表作。

### 5. AGILE —《AGILE: A Novel Reinforcement Learning Framework of LLM Agents》
- **会议**：NeurIPS 2024　|　**引用**：49　|　**代码**：https://github.com/bytarnish/AGILE

**核心思想（大白话）**：把"做一个 agent"整个过程**直接当成一个 RL 问题端到端地训**。agent 不只是会聊天，还内置了**记忆、工具、反思、以及"不会就问专家"** 的能力，这些全都一起用 RL 学出来。

**怎么做**：让 **LLM 本身充当 RL 里的"策略（policy）"**，整套系统（记忆模块、工具调用、反思、向人类/更强 LLM 求助）用 **PPO 端到端训练**。其中"主动求助专家"是亮点——agent 学会"什么时候该自己干、什么时候该问"。

**实验怎么做**：在 **ProductQA（电商问答）、MedMCQA（医学选择题）、HotPotQA（多跳问答）** 上测；关键是把 GPT-4 也放进**同一个 AGILE 框架**里做公平对比。

**主要结果**：基于 **7B、13B 开源模型 + PPO 训练的 agent，超过了用 GPT-4 的 agent**（如 ProductQA 上 13B 版 0.784 > GPT-4 版 0.747；MedMCQA 85.2% 超过此前 SOTA）。说明**小模型靠 RL + 好框架能打赢大模型**。

**为什么重要**：提供了一个"记忆+工具+反思+求助"四件套、端到端 RL 训练的完整 agent 框架范式。

## 四、2025 年 前五（按引用排，含领域标杆）

### 🥇 1. Search-R1 —《Search-R1: Training LLMs to Reason and Leverage Search Engines with RL》
- **会议**：COLM 2025（arXiv:2503.09516）　|　**引用**：**1068（本领域引用王）**　|　**代码**：https://github.com/PeterGriffinJin/Search-R1（约 4.9k stars）

**核心思想（大白话）**：让模型边想边**自己上网搜**。模型在推理过程中，遇到不确定的就**自动发起搜索→读结果→接着想→可能再搜**，多轮交错，最后给答案。整个"何时搜、搜什么"的能力用 RL 训出来。

**怎么做**：
- 用 RL（PPO/GRPO）训练，**奖励就是最终答案对不对**（结果导向，RLVR）；
- 关键技巧：**对"检索回来的文本"做 token 掩码**，训练时不让模型为搜索引擎返回的内容"背锅"，使训练稳定；
- 支持多轮"思考↔检索"交错。

**实验怎么做**：在 **7 个问答数据集**（NQ、TriviaQA、HotpotQA、2WikiMultiHopQA 等，含单跳和多跳）上测；基座用 Qwen2.5-3B/7B；对比 RAG（检索增强）、IRCoT、以及不带搜索的 RL 等基线。

**主要结果**：相比这些基线有显著提升（7B 模型相对提升幅度可观），且模型自己"学会了"反复检索、验证、纠错的行为。

**为什么重要**：它是"搜索/研究类 agent（Deep Research 那一挂）"用 RL 训练的奠基之作，引用最高，开源也最火。**强烈建议第一篇精读。**

### 🥈 2. UI-TARS —《UI-TARS: Pioneering Automated GUI Interaction with Native Agents》
- **会议**：arXiv 2025（字节跳动）　|　**引用**：449　|　**代码**：https://github.com/bytedance/UI-TARS（模型）、https://github.com/bytedance/UI-TARS-desktop（桌面应用，约 36k stars）

**核心思想（大白话）**：做一个"**纯看屏幕截图就能操作电脑/手机**"的端到端 agent——不靠读网页 HTML 或辅助接口，直接像人一样"看界面→决定点哪→执行"。其中用 RL 让它"**先想清楚再动手**"（System-2 慢思考）。

**怎么做**：基于强视觉-语言模型（Qwen2-VL 架构），统一"感知截图 + 推理 + 输出动作"；引入 **RL 强化推理**（动手前先生成思考），并支持**反思式在线学习**（从自己的操作历史里改进），还利用推理时扩展（inference-time scaling）。

**实验怎么做**：在一大批 GUI/电脑操作基准上测，如 **OSWorld、AndroidWorld、WebArena、以及多个 GUI grounding 基准**。

**主要结果**：在这些基准上达到 SOTA，**超过 GPT-4o、Claude 的"电脑操作（computer-use）"能力**。后续还迭代了 UI-TARS-1.5 / UI-TARS-2。

**为什么重要**：是"电脑/手机操作 agent（computer-use）"方向开源最有影响力的工作之一，工业界落地度高。

### 🥉 3. ReTool —《ReTool: Reinforcement Learning for Strategic Tool Use in LLMs》
- **会议**：arXiv 2025（字节跳动 Seed）　|　**引用**：297　|　**代码**：https://github.com/ReTool-RL/ReTool

**核心思想（大白话）**：教模型在解题（尤其数学）时，**该用代码就用代码**——在自然语言推理过程中**实时插入"写代码→执行→看结果→继续推理"**，而且"什么时候该调用代码工具"是用 RL 自己学出来的，不靠人定规则。

**怎么做**：
- **把实时代码执行嵌入推理过程**（边想边跑代码，结果回填继续想）；
- **自动化的结果奖励 RL**：允许在 rollout 里多轮真实执行代码，仅凭"最终对不对"来教模型何时/如何调工具。

**实验怎么做**：在数学竞赛基准 **AIME 2024/2025** 上测；对比"只用文字推理的 RL"基线。

**主要结果**：**ReTool-32B 仅用 400 步训练就达到 67% AIME 准确率**，而纯文字 RL 跑 1080 步才 40%——**又快又强**。还观察到模型自发学会"代码自我纠错"等高级行为。

**为什么重要**：是"工具集成推理（TIR）"用 RL 训练的标杆，证明"会用代码"能大幅提升推理上限。

### 4. ToolRL —《ToolRL: Reward is All Tool Learning Needs》
- **会议**：NeurIPS 2025　|　**引用**：262　|　**代码**：https://github.com/qiancheng0/ToolRL　|　UIUC

**核心思想（大白话）**：教模型用工具，**关键不在于喂多少示范，而在于"奖励怎么设计"**。这篇是第一篇系统研究"工具使用的 RL 奖励设计"的工作。

**怎么做**：用 RL（GRPO/PPO）训练，把"用对工具"这件事的奖励**拆成三块**精细打分：
1. **工具名**选对了吗；
2. **参数名**对不对；
3. **参数值**对不对。
（而不是简单地"全对才给 1 分、否则 0 分"，让学习信号更密、更好学。）

**实验怎么做**：在多种工具调用基准上测，覆盖不同任务类型和复杂工具集；对比基座模型、SFT（监督微调）模型。

**主要结果**：相比基座模型**提升约 17%**、相比 SFT 模型**提升约 15%**；并论证了 **SFT 在"没见过的新工具"上泛化差，而 RL（类 R1 风格）泛化更好**。

**为什么重要**：把"工具学习"从"堆数据"引向"设计奖励"，为通用工具 agent 提供了可复用的奖励配方。

### 5. RAGEN —《RAGEN: Understanding Self-Evolution in LLM Agents via Multi-Turn RL》
- **会议**：arXiv 2025　|　**引用**：229　|　**代码**：https://github.com/RAGEN-AI/RAGEN

**核心思想（大白话）**：不是又提一个"刷榜"方法，而是**认真研究"多轮 agentic RL 为什么训着训着就崩了"**，并给出稳定训练的配方。提出训练框架 **StarPO**（State-Thinking-Actions-Reward Policy Optimization）。

**怎么做**：
- **StarPO**：把"状态→思考→动作→奖励"整条多轮轨迹作为优化对象的通用框架；
- 发现一种典型不稳定现象 **"Echo Trap（回声陷阱）"**——模型陷入重复套路、奖励方差塌缩、推理多样性消失；
- 提出 **StarPO-S（稳定版）**：用过滤高不确定性样本、控制梯度等手段稳住训练。

**实验怎么做**：刻意在**极简、可控的环境**（如 Bandit 多臂老虎机、Sokoban 推箱子、Frozen Lake 冰湖）上做，方便把"训练动力学"看清楚，而不是被复杂任务干扰。

**主要结果**：揭示了多轮 agentic RL 的不稳定根因，给出 StarPO-S 让训练更稳、推理更不易退化的实证结论。

**为什么重要**：偏"方法论/原理"，是想**自己动手训 agent 的人**避坑必读——告诉你训练会怎么崩、怎么救。
